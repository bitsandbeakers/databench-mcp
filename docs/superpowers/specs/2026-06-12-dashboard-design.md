# Dashboard Generation — Design Spec

**Date:** 2026-06-12
**Project:** databench-mcp
**Status:** Approved

---

## Overview

Adds a presentation layer to databench-mcp. `build_dashboard` walks a project's chart sidecars, exports each table to parquet, and generates a standalone `dashboard.py` (Dash app) with one tab per dataset and a Findings tab. The generated app is self-contained and targets Dash Community Cloud for free public deployment.

`EXPECTED_TOOL_COUNT` bumps from 21 → **22**.

---

## 1. Architecture

### Layer separation (unchanged pattern)

```
core/dashboard.py   →  build_dashboard logic
tools/dashboard.py  →  MCP tool wrapper
server.py           →  register tool; EXPECTED_TOOL_COUNT = 22
```

### Output files

`build_dashboard(project)` writes to `workspace/{project}/dashboards/`:

```
dashboards/
  dashboard.py          ← standalone Dash app
  requirements.txt      ← dash, plotly, pandas (pinned)
  DEPLOY.md             ← dash deploy instructions
  data/
    {table}.parquet     ← one parquet per project table
```

### Workspace change

`"dashboards"` added to `_SUBDIRS` in `workspace.py` so the directory is scaffolded on `project_create`.

## 2. `build_dashboard` Tool

### Contract

```python
build_dashboard(project)
→ {
    "dashboard_py": str,       # absolute path to dashboard.py
    "requirements_txt": str,   # absolute path to requirements.txt
    "deploy_md": str,          # absolute path to DEPLOY.md
    "tabs": int,               # number of dataset tabs generated
    "charts_embedded": int,    # total charts included
    "tables_exported": [str],  # table names written to parquet
    "warning": str | None      # e.g. "dashboard.py already existed and was overwritten"
  }
```

### Algorithm

1. Read `manifest.json` → list of table names (`datasets` keys)
2. Read all `charts/*_params.json` sidecars → group by `table` field; sort each group by timestamp
3. Read `findings.yaml` → all findings
4. Export each table from DuckDB to `dashboards/data/{table}.parquet` via `conn.execute('SELECT * FROM "{table}"').df().to_parquet(...)`
5. Generate `dashboard.py` (see shape below)
6. Write `requirements.txt` and `DEPLOY.md`
7. Return result dict

### Generated `dashboard.py` shape

```python
# Generated by databench-mcp build_dashboard — do not edit manually.
# Deploy: pip install dash && python dashboard.py  OR  dash deploy
import dash
from dash import dcc, html, dash_table
import plotly.express as px
import plotly.graph_objects as go
import pandas as pd
from pathlib import Path

DATA_DIR = Path(__file__).parent / "data"

# --- data loading ---
tables = {
    "providers": pd.read_parquet(DATA_DIR / "providers.parquet"),
    "edges": pd.read_parquet(DATA_DIR / "edges.parquet"),
}

# --- chart helpers ---
def _make_figure(chart_type, df, columns, params):
    """Render a Plotly figure from a DataFrame. Mirrors core/viz.py logic."""
    # ... per-chart-type rendering logic (histogram, boxplot, scatter, etc.) ...

# --- layout ---
app = dash.Dash(__name__)

dataset_tabs = [
    dcc.Tab(label="providers", children=[
        html.Div([
            dcc.Graph(figure=_make_figure("histogram", tables["providers"], ["charge"], {})),
            dcc.Graph(figure=_make_figure("scatter", tables["providers"], ["charge", "payment"], {})),
        ], style={"display": "grid", "gridTemplateColumns": "1fr 1fr", "gap": "16px"}),
    ]),
    dcc.Tab(label="edges", children=[
        html.Div([
            dcc.Graph(figure=_make_figure("network_graph", tables["edges"], ["src", "tgt"], {"layout": "spring"})),
        ], style={"display": "grid", "gridTemplateColumns": "1fr 1fr", "gap": "16px"}),
    ]),
]

findings_data = [
    {"id": "f001", "method": "kmeans", "table": "providers", "summary": "3 clusters identified", "metrics": "inertia=412"},
    # ...
]

findings_tab = dcc.Tab(label="Findings", children=[
    dash_table.DataTable(
        data=findings_data,
        columns=[{"name": c, "id": c} for c in ["id", "method", "table", "summary", "metrics"]],
        style_table={"overflowX": "auto"},
    )
])

app.layout = html.Div([
    dcc.Tabs(children=dataset_tabs + [findings_tab])
])

if __name__ == "__main__":
    app.run(debug=False)
```

Importable as a library AND runnable as a script. Standalone dependency: `pip install dash plotly pandas pyarrow`.

### Chart rendering constraints

`_make_figure` operates on DataFrames loaded from parquet. Charts that require binary finding artifacts are **skipped with a warning** in v1:

| Chart type | v1 status |
|---|---|
| `histogram`, `boxplot`, `scatter`, `scatter_matrix`, `correlation_heatmap` | Rendered |
| `network_graph` (no `finding_id`) | Rendered |
| `cluster_scatter`, `shap_beeswarm`, `feature_importance_bar` | Skipped — requires `.npy` artifact |
| `network_graph` with `finding_id` | Skipped — requires `_communities.json` artifact |

Skipped charts are noted in the `warning` field of the return value.

### Generated `requirements.txt`

```
dash==2.17.1
plotly==5.22.0
pandas==2.2.2
pyarrow==16.1.0
```

Versions pinned to the project's test environment at generation time.

### Generated `DEPLOY.md`

```markdown
# Deploy to Dash Community Cloud

1. Install the Dash CLI: `pip install dash`
2. Log in: `dash login`
3. From this directory: `dash deploy`

The app will be live at https://your-username.pythonanywhere.com/<project-name>
```

### Error conditions

| Condition | Behaviour |
|---|---|
| No charts in `charts/` | `ValueError: no charts to embed — run create_chart first` |
| Table in sidecar missing from DB | warning returned; chart skipped |
| `dashboard.py` already exists | overwritten; warning returned |

## 4. Testing Strategy

```
tests/
  test_core_dashboard.py           # unit tests
  tools/
    test_dashboard_integration.py  # integration test
```

### `test_core_dashboard.py`

- `test_build_dashboard_generates_files` — build on project with 2 charts across 2 tables; verify `dashboard.py`, `requirements.txt`, `DEPLOY.md`, and both parquet files exist
- `test_build_dashboard_tab_count` — result `tabs` equals number of distinct tables with charts
- `test_build_dashboard_charts_embedded` — result `charts_embedded` equals number of renderable sidecars
- `test_build_dashboard_no_charts_raises` — no `_params.json` sidecars → `ValueError`
- `test_build_dashboard_overwrites_existing` — call twice; second call returns warning, file updated

### `test_dashboard_integration.py`

1. Ingest synthetic CSV → `profile_table` → `create_chart` on two different tables
2. `build_dashboard` → verify all output files written
3. Verify `dashboard.py` is valid Python (compile with `py_compile`)
4. Verify parquet files are readable with `pd.read_parquet`
5. Verify `tabs == 2` (one per table) and `charts_embedded >= 2`
